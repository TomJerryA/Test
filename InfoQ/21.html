Microsoft Office 开发者工具支持搜索配置模板、应用验证和Azure调试

Microsoft最近发布了Office Developer Tools for Visual Studio 2012 ，该工具包含新的项目和搜索配置项目模板，新增的App for Office项目允许用户定义自己的应用并将其绑定到一个文档，同时新创建的应用能够作为一个文档库的默认模板用于自己的项目。

借助于这些新特性，开发者能够使用内置向导基于文档库类型创建客户化的列表。此外，App for Office还包含一个新功能，通过这个新功能你能够在发布过程中验证自己的应用。

如果你在publish窗口开启了Validate my app for the Office Store 选项，那么工具将会自动地对你应用中的一些常规检查项进行验证，同时会显示详细的检查结果，告诉你那些可能导致应用无法成功提交到Office商店的问题的列表。

Microsoft Office Developer Tools for Visual Studio 2012还提供了使用Windows Azure服务总线调试远程事件接受者的功能。为了使用该功能，你仅需要登录Windows Azure管理门户，添加一个新命名空间，使用Access Key按钮发现服务总线端的连接字符串，并将该字符串粘贴到Properties|SharePoint标签页，然后添加一个断点并按F5键即可。

“Office Developer Tools for Visual Studio 2012提供了广泛且深入的支持，包括新模板、新设计器以及大量的一体化生命周期工具，”Microsoft公司负责开发的副总裁S. Somasegar说。

开发者能够借助于Microsoft Office Developer Tools for Visual Studio 2012使用Visual Studio 2012为Office 2013、Office 365和SharePoint 2013 创建丰富的、引入注目的应用。它还提供了一个持续集成工作流，支持本地的Team Foundation Server和云托管的Team Foundation服务，能够为Office和SharePoint应用构建工作流。
