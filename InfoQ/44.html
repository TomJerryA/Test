Azul推出OpenJDK 6和7的商业化版本，适用于Windows和Linux

Zulu是OpenJDK的免费版本。Azul Systems已经宣布，现在， Zulu在基于Linux和Windows的部署上支持Java 6和Java 7。他们会通过Zulu社区支持论坛对其提供支持。此外，该公司正在推广Zulu Enterprise，这是OpenJDK的一个商业支持版本。

Zulu支持多种Linux和Windows版本——Red Hat Enterprise Linux、SUSE Linux Enterprise Server、Ubuntu Desktop and Server以及CentOS——并计划将来根据社区反馈支持其它发行版。Azul此前曾经宣布Zulu支持Windows Azure，如今又宣布了对本地Windows部署的更多支持。目前，Zulu支持的操作系统和平台的完整列表如下：

  • Red Hat Enterprise Linux 5.2 或更高版本和6.0或更高版本
  • SUSE Linux Enterprise Server 11 sp1、sp2和spa
  • CentOS 5.2或更高版本和6.0或更高版本
  • Ubuntu 10.04 LTS和12.04 LTS
  • Windows Server：2008、2008 R2、2012和2012 R2
  • Windows Desktop：7、8和8.1
  • Hypervisors：VMware、Hyper-V、KVM
  • Cloud：Amazon AWS、Windows Azure、Rackspace
  • JDK版本：Java SE 6、7

据Auzl营销副总裁Howard Green说，他们会在“OpenJDK 8正式版发布后很快”就提供对它的支持。由于在Java 6以前没有OpenJDK，所以他们不会提供对早期版本（如Java 5）的支持。

Zulu独立于Azul的Zing产品，完全基于OpenJDK代码库。Azul systems首席技术官Gil Tene说：

我们会积极地解决客户的问题和修复bug，而所有这些将直接上报至OpenJDK项目组，因为我们将努力工作，避免修改OpenJDK的代码造成其分化。

与社区支持的版本一起，Azul正在推广Zulu Enterprise，这是OpenJDK的一个商业化的完全支持版本。从最初的正式版开始，Java主版本号会获得10年的支持（例如，如果是Java 6，则Zulu Enterprise支持合同直到2024年都适用）。

Azul提供了三个层次的订阅型支持选项（黄金级、白金级、精英级）。精英级选项包含“应用程序守护（Application Guard）”，允许客户具体指定一定数量的操作系统、JVM版本和配置信息，这些内容其后会包含到Azul的质量保证过程之中。从而，Zulu JDK/JVM可以针对企业客户使用的具体操作系统和JVM组合进行测试。

Zulu Enterprise客户还可以选择专用客户技术经理选项。该选项会为客户提供一名专用技术支持工程师，以保证企业内Zulu使用知识的连续性和参与实际的案例管理。

Tene告诉我们，OpenJDK的商业支持是Azul客户一段时间以来的要求，Zing也是如此。他认为，如果Oracle更明确地表示完全不支持Java的免费版本，那么Java付费支持市场将会扩大：

Java是一款产品。Oracle提供完全没有支持的免费下载，如果用户想要商业支持就需要购买。商业支持的版本既包括生命周期业已终结的版本，如Java 6和Java 5，也包括当前正在推广的不付费就不支持的版本。所以，用户可以下载三天前出来的最新免费更新51,这很棒，那是他们的善意——不过，如果你遇到了问题，最好不要向他们求助。

截至今日，Azul产品Zing的商业支持是现有商业支持中最便宜的。Zulu的商业支持“大致也是同样的价格，但会比Zing更便宜，”Azul Systems营销副总裁Howard Green如是说：

定价可以商洽，但黄金级支持的起步价为每台服务器每年400美元，享受总价折扣和更高级支持。

Oracle和IMB也都为Java企业客户提供付费支持选项。Oracle的起步价为每个处理器250美元，40个处理器起订（1万美元）。了解更多信息，请点击这里。
