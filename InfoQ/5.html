Visual Studio 2012 Update 2最终预览版发布

在正式发布VS 2012 Update 2版本前，微软一直定期发布预览版本，秉持这一作风，微软此次发布了CTP4。这是Update 2的第一个“go-live”(译者注：go-live指的是将测试中的产品加入生产环境的许可)状态的版本，这意味着它已获得微软的全面支持（请注意下文中TFS 2012 go-live和VS 2012 go-live之间的区别）。这将是在正式发布Update 2之前的最后一个社区技术预览版本。

CTP4 继续围绕Update 2的主题——“现代应用生命周期和现代应用开发”进行了全面的改进。为了达到这个目标，首先发布的功能是LightSwitch HTML 客户端的更新预览版本。首席项目经理John Stallo指出这次发布的两个主要目标是支持创建移动端、跨平台的web客户端以及将程序发布到SharePoint 2013应用目录。

VS 2012在基于浏览器的测试管理方面也得到了进一步的改进，目前已可以对结果文本进行编辑并支持将多个测试用例步骤进行内联，而不需要在被测环境上安装软件。

微软的Brian Harry也指出备份和恢复工具（之前提供单独下载）将会整合到Update 2中。TFS的系统管理员将会从中受益，因为可以在升级后依然保留旧有的TFS配置。

如果需要Git支持功能的开发者仍需等待，因为适用于CTP 4版本的Git扩展尚未发布。Harry指出该扩展将“在接下来的一到两天”发布。这部分的功能支持并不是Update 2的一部分，仍然需要用户进行单独下载，同时Harry指出这种方式将使微软可以更快地发展该功能。

需要注意的是CTP 4的go-live状态目前只适用于Team Foundation Server 2012，而并不适用于VS 2012，因此VS 2012的CTP4 版本并不适用于生产环境。
