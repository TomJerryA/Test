Java EE 8开始最后一轮社区调研

Oracle宣布开始第三轮社区调研，这也是最后一轮，邀请Java EE社区就Java 企业版的下一版本Java EE 8以及Glassfish参考实现的成形发表意见。

前面两轮调研分别于2013年12月和2014年1月宣布。

根据Oracle的Java EE/GlassFish布道师Reza Rahman在The Aquarium博客上发布的信息，

“前面两轮调研关注的是具体的特性，而最后这轮探讨的完全是确定最重要特性的优先级。”

调研的引言中加入了这么一句：

“我们无法做到在一次发布中实现列表上的一切特性，而且肯定还有些是需要做的，但没有列出来。希望你们能告诉我们，如何分配有限的资源。”

Oracle分析了前两轮调研的结果，社区最喜欢的一些特性包括：

  • 79%的人认为包含Java API for JSON Binding的JSR非常重要
  • 超过77%的人提议标准化服务端事件Java API
  • 79%的人要求Java EE 8中的安全拦截器
  • 相对于JMX或其他方案，超过77%的人更喜欢RESTful风格的部署、管理和监控API
  • 74%要求被删减的EJB 2.x远程和本地接口

有40%的人说现在还不是关注NoSQL标准化的时候，让人有点吃惊。

新一轮调研只是要求受访者跨13个大类对各条目打分，总分为100。这些大类包括“JCACHE”、“Java API for JSON Binding”以及“日志中的改进”。

最后一轮调研将于太平洋时间2014年3月24日星期一的中午12点关闭。
