BizTalk服务的发布标志着Microsoft集成平台转向“云优先”的策略

经过了长期的孵化，Microsoft终于发布了自己的云集成工具——Windows Azure BizTalk服务。在反复重申自己对本地集成工具承诺的同时，Microsoft也已经表示他们首先会推动云平台的创新。

Microsoft公司的副总裁Guthrie上周在一篇博客文章中宣布了Windows Azure BizTalk 服务的发布。他为那些想要使用云托管消息引擎的组织解释了该服务的核心EAI和B2B用例。

Windows Azure BizTalk服务支持强大的业务场景，例如供应链、基于云计算的电子数据交换以及企业应用程序集成，所有这一切都使用了熟悉的工具集并提供企业级的可靠性。它为合伙伙伴之间的EDI关系管理、与本地资源建立EAI桥梁提供了内置的支持，同时还为与本地SAP、SQL Server、Oracle和Siebel系统的集成提供了内置支持。你还能够选择将Windows Azure BizTalk服务与本地BizTalk Server部署集成——能够构建强大的混合企业解决方案。

发布的产品包含一系列的输入和输出适配器、称为“桥梁（Bridges）”的可执行消息模型、一个用于贸易伙伴管理的门户和扩展集成应用程序的基础设施。开发者可以在Microsoft Visual Studio中构建消息解决方案，然后在运行时将组件部署到Windows Azure上。该订阅服务有四个可用的版本：开发版、基础版、标准版和高级版。开发版适用于MSDN订阅用户，提供了该服务的所有核心能力，但是不包含扩展和备份解决方案的能力。后面的版本支持更多的“扩展单元”，另外还包括本地连接、备份和恢复以及数据归档功能。价格范围是从开发版每小时$0.13到高级版每小时（每单元）$8.06。

BizTalk Services最初的预览版发布于2011年，那时候它的名字还是Windows Azure Service Bus EAI Labs，现在它已经与成熟的本地Microsoft集成产品BizTalk Server松耦合。虽然它们共享一个名字，但是云服务仅仅包含BizTalk Server功能的一个很小的子集，为其中某一个构建的解决方案不能运行在另一个上面。但是最近在西雅图举行的一个BizTalk高层会议上Microsoft分享了它们针对BizTalk服务的侵略性计划，该计划可能最终会交付与它同名产品的等值服务。他们期望通过引入新的适配器弥补某些直接的差距，让第三方能够更简单地开发适配器，同时为B2B场景添加EDIFACT支持。未来更加野心勃勃的功能包括持久的工作流组件、业务流程设计器、规则设计器和业务活动监控，并会改善管理能力。

Microsoft打算至少每三个月更新一次BizTalk服务。这和BizTalk Server的更新节奏形成了鲜明的对比，后者每两年才会发布一个主要的新版本。Microsoft声明说新技术在最终包含进本地产品之前首先会推向云服务。如果你想获取更多的信息可以访问BizTalk服务的产品页面，同时还能够下载开发者SDK体验该服务。
